Reading Assignment 1: Cheat Sheet

Word Wide Web General

1. What are most essential technologies that made the World Wide Web such a success?
HTTP (Hypertext Transfer Protocol, Hypertext Markup Language (HTML), Uniform Resource Locator (URLs)

2. How are resources (html pages, images, etc) addressed in the web?
URL is the most common form of resource identifier.

3. What are the general components of an URL?
Five Components: Scheme, Authority, Path, Query, Fragment
Three Subcomponents: Userinfo, Host, Port

Web Server

1. What is a web server?
A web server is a computer that stores web server software and a website's component files

2. What does a web server do?
A web server uses HTTP and other protocols to respond to client requests made over the World Wide Web.

3. How can content be stored on a web server?
Content can be stored as static content or as dynamic content

4. How is content in webpages described?
Web content refers to the textual, aural or visual content published on a website.

Web Client

1. What is a web client?
A Web client typically refers to the Web browser in the user's machine or mobile device.

2. What does a web client do?
A web client connects to a web server over HTTP.

3. Is the Web Client and Web Browser the same?
No, Web Browser is only a part of the Web Client.
A Web Browser delivers Web pages received from the Web Server to the Web Client.

Reading Assignment 2: Cheat Sheet

HTML

1. Name the components of html elements properly
An HTML element is defined by a start tag, some content, and an end tag.
It is everything from the start tag to the end tag.

2. Identify void elements and how they are denoted
HTML elements with no content are called void (or empty) elements.
Example: <br>, defines a line break.

3. Identify attributes

  • All HTML elements can have attributes
  • Attributes provide additional information about elements
  • Attributes are always specified in the start tag
  • Attributes usually come in name/value pairs like: name="value"

4. Write down the correct <DOCTYPE> declaration for html 5
<!DOCTYPE html>

5. Write down syntactically correct statements for the following element

  • Article <article>
    • The <article> tag specifies independent, self-contained content.
    An article should make sense on its own and it should be possible to distribute it independently from the rest of the site.

  • Body <body>
    • The <body> tag defines the document's body.
    The <body> element contains all the contents of an HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.

  • Line break <br/>
    • Use <br> if you want a line break (a new line) without starting a new paragraph

  • Headings <h1>, ...
    • The <h1> to <h6> tags are used to define HTML headings.
    <h1> defines the most important heading.
    <h6> defines the least important heading.

  • Section with meta-information <head>
    • The <head> element is a container for metadata (data about data) and is placed between the <html>tag and the <body> tag.
    Metadata is data about the HTML document. Metadata is not displayed.

  • Top level element <html>
    • The <html> element is the root element and it defines the whole HTML document.

  • Image named Team.jpg <img src="Team.jpg">
    • The <img> tag is used to embed an image in an HTML page.
    Images are not technically inserted into a web page; images are linked to web pages.
    The <img> tag creates a holding space for the referenced image.
    The <img> tag has two required attributes:
    src - Specifies the path to the image
    alt - Specifies an alternate text for the image, if the image for some reason cannot be displayed

  • Paragraph <p>
    • A paragraph always starts on a new line, and browsers automatically add some white space (a margin) before and after a paragraph.

  • Section <section>
    • The <section> tag defines a section in a document.

  • Label appearing in the title bar <title>
    • The title attribute defines some extra information about an element.
    The value of the title attribute will be displayed as a tooltip when you mouse over the element

  • Link to other pages <a>
    • The <a> tag defines a hyperlink, which is used to link from one page to another.
    The most important attribute of the <a> element is the href attribute, which indicates the link's destination.
    By default, links will appear as follows in all browsers:
    An unvisited link is underlined and blue
    A visited link is underlined and purple
    An active link is underlined and red

  • Comment
    • The comment tag is used to insert comments in the source code.
    Comments are not displayed in the browsers.

  • Head with title and meta elements
    • The <title> element defines the title of the document.
    The title must be text-only, and it is shown in the browser's title bar or in the page's tab.
    The <meta> element is typically used to specify the character set, page description, keywords, author of the document, and viewport settings.

CSS

1. Name the components of a css rule properly
A CSS rule consists of a selector and a declaration block.


The selector points to the HTML element you want to style.
The declaration block contains one or more declarations separated by semicolons.
Each declaration includes a CSS property name and a value, separated by a colon.
Multiple CSS declarations are separated with semicolons, and declaration blocks are surrounded by curly braces.

2. Use combinators properly: direct child, descendant, adjacent sibling, sibling

  • descendant selector (space)
  • child selector (>)
  • adjacent sibling selector (+)
  • general sibling selector (~)

3. Specify colors by color names and in hexadecimal notation

  • Tomato = #FF6347
  • Orange = #FFA500
  • Gray = #3CB371

4. Declarations for color, background color
style="color:Tomato;"
style="background-color: Tomato;"

5. Pseudo classes for the <a> element 

a:link {            Note: a:hover MUST come after a:link and a:visited in the CSS definition in order to be effective.
    color: red;     Note: a:active MUST come after a:hover in the CSS definition in order to be effective.
  }
                            
a:visited {
    color: green;
  }
                            
a:hover {
    color: hotpink;
  }
                            
a:active {
    color: blue;
  }

6. Declaration for background image 

body { 
    background-image: url("image.name"); 
  }

7. Declaration for text alignment (left, right, center, justify)
text-align: center/right/left/justify;

  • center: display in center/li>
  • right: display on the right
  • left: display on the left
  • justify: each line is stretched so that every line has equal width, and the left and right margins are straight (like in magazines and newspapers)

8. Declaration for text decoration (overline, underline, line-through)
text-decoration-line: overline/underline/line-through

  • overline: line over text
  • underline: line below text
  • line-through: line through text
  • combinations: you can combine the 3 lines

9. Declaration for text transformation (uppercase, lowercase, capitalize)
text-transform:: uppercase/lowercase/capitalize

  • uppercase: every letter is uppercase
  • lowercase: every letter is lowercase
  • capitalize: every first letter of a word is uppercase

10. Identify the difference between serif and sans-serif fonts
1. Serif fonts have a small stroke at the edges of each letter. They create a sense of formality and elegance.
2. Sans-serif fonts have clean lines (no small strokes attached). They create a modern and minimalistic look.

11. Declarations for font families
(Generic Font Family)
font-family: serif/sans serif/monospace/Cursive/Fantasy
OR
(Examples of Font Names)
font-family: "Lucida Console" / "Times New Roman" / "Arial"

12. Declarations for font style normal and italic
Font-style: normal/italic/oblique

  • Normal: normal
  • Italic: italic
  • oblique: “leaning” (similar to italic)

13. Declarations for font sizes
font-size: px/em/%/vw

  • px: normal
  • em: italic
  • %: percentage (100% is normal)
  • vw: “viewport width”, Responsive Font Size, text size will follow the size of the browser window

14. Declarations for font weights
font-weight: normal/lighter/bold

  • Normal: normal
  • Lighter: light
  • Bold: thick
